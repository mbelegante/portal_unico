# Portal Único do Cidadão Brasileiro

Portal governamental fictício criado para demonstrar, de forma proposital, uma péssima experiência de usuário. O projeto simula a solicitação de um benefício social com fluxo confuso, burocrático e frustrante, mas ainda funcional.

## Problema

O problema escolhido foi a má experiência do usuário em sistemas digitais burocráticos, especialmente em portais de serviços públicos. Muitas vezes, pessoas que precisam solicitar benefícios, documentos ou atendimentos online encontram interfaces confusas, mensagens pouco claras, excesso de etapas, baixa acessibilidade e navegação frustrante.

O público impactado são cidadãos que dependem de serviços digitais governamentais, principalmente pessoas com pouca familiaridade com tecnologia, idosos, trabalhadores com pouco tempo disponível e usuários que precisam concluir processos importantes sem suporte técnico.

## Solução

Foi desenvolvido um MVP de um portal fictício chamado “Portal Único do Cidadão Brasileiro”, simulando a solicitação de um benefício social.

A proposta não é criar uma boa interface, mas sim uma interface propositalmente ruim, inspirada em experiências como o User Inyerface. O sistema possui várias escolhas de UX/UI problemáticas, como:

* Barra de progresso enganosa;
* Botões com cores inconsistentes;
* Botão “Próximo” que muda de posição;
* Etapa em que o botão “Voltar” faz o usuário avançar;
* Etapa em que clicar em “Próximo” leva o usuário de volta ao início;
* Campos obrigatórios que só aparecem depois da tentativa de envio;
* Mensagens de erro vagas;
* Pop-ups aleatórios;
* Temporizador fictício;
* Checkbox obrigatório escondido;
* Confirmação final ambígua e burocrática.

Apesar da experiência ruim, o fluxo ainda permite que o usuário chegue até a tela final.

## Como testar

1. Acesse o link do MVP ou abra o arquivo `index.html` no navegador.
2. Preencha as etapas do formulário.
3. Observe os comportamentos ruins propositalmente implementados, como mensagens vagas, pop-ups, barra de progresso enganosa e botões inconsistentes.
4. Na etapa 3, tente clicar em “Próximo” e veja que o sistema retorna ao início.
5. Para avançar na etapa 3, clique em “Voltar”.
6. Continue preenchendo o fluxo até chegar à tela final de conclusão.

## Validação

Foram realizados testes manuais durante o desenvolvimento para verificar se o fluxo continuava funcional, mesmo com a experiência propositalmente ruim.

Durante os testes, foram observados os seguintes pontos:

* O usuário consegue preencher o formulário até o final;
* A barra de progresso funciona, mas apresenta um comportamento enganoso;
* A etapa 3 cria confusão ao inverter a lógica dos botões;
* Os pop-ups aparecem durante o uso e interrompem o preenchimento;
* Os campos obrigatórios aparecem somente depois da tentativa de envio;
* A tela final é exibida corretamente após a conclusão do fluxo.
